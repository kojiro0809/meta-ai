##  開発の背景：「AI × 金融」への知的好奇心と技術的探求
**「AIは、複雑怪奇な金融市場の未来をどう捉えるのか？」**
本プロジェクトは、そんな純粋な知的好奇心からスタートしました。昨今盛り上がりを見せるFinTech領域における「AIと金融のコラボレーション」に強い魅力を感じ、その予測メカニズムを自分自身の手で実装し、解き明かしてみたいと考えました。

そこで、実践的な時系列予測を学ぶための題材として、Meta社の『Prophet』を選定しプロトタイプ開発に着手しました。

### 「ブラックボックス」からの脱却と技術的責任
開発当初は、生成AIを活用して爆速で構築したものの、内部の計算ロジックがブラックボックスであり、予測結果に対する「説明責任」を果たせていないことに違和感を覚えました。
そこで、時系列解析の数学的背景を改めて学習し、**Prophetが「加法モデル（Additive Model）」に基づく統計モデルであること**を理解した上で、金融データの特性に合わせたパラメータチューニングとコードのリファクタリングを実施しています。

## 🧠 Prophetアルゴリズムの理解
Prophetの予測は、以下の4つの要素の足し算によって構成されています。
$$y(t) = g(t) + s(t) + h(t) + \epsilon_t$$

- **$g(t)$ (Trend):** 大局的な成長または衰退の方向性
- **$s(t)$ (Seasonality):** 曜日や季節ごとの周期的な変動
- **$h(t)$ (Holiday/Event):** 突発的なイベント（半減期や祝日など）の影響
- **$\epsilon_t$ (Error):** モデルで説明できないランダムなノイズ

この「人間が解釈しやすい（Explainable AI）」という特性を活かし、本アプリでは単なる予測結果の提示だけでなく、**「トレンド」と「周期性」を分解して表示する機能**を実装しました。

## 実装における改善点（AIコードからのリファクタリング）

### 1. 金融データにおける過学習（Overfitting）の防止
- **課題:** 初期コードではデフォルト設定（`Prophet()`）で学習を行っていたため、金融データの激しいノイズに過剰に反応し、未来予測の精度が不安定になるリスクがありました。
- **解決策:** `changepoint_prior_scale` を調整してトレンドの変化を適度に平滑化。また、週単位や年単位の周期性を明示的に考慮しつつ、日次の微細なノイズを無視する設定を行うことで、より堅牢な（Robustな）モデルへとチューニングしました。

### 2. データ取得・学習プロセスの堅牢化
- **課題:** 存在しないティッカーの入力や、APIの通信エラーによってアプリ全体がクラッシュする問題。
- **解決策:** `try-except` 構文による例外処理を徹底。データフェッチからAIの学習フェーズまで、各ステップでエラーを捕捉し、ユーザーに適切なフィードバックを返すよう設計を見直しました。

##  主な機能
- **リアルタイム推論:** `yfinance` APIから最新の市場データを取得し、即座に3年後の価格をシミュレーション。
- **多角的な分析:** 予測価格の「中央値」だけでなく、統計的な「上限・下限」のレンジを表示し、リスク管理の視点を提供。
- **トレンド分解:** 資産価格が「なぜその方向に動いているのか」を成分ごとに可視化。
